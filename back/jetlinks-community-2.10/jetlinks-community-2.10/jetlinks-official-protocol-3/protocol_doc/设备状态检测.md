JetLinks默认查看设备详情时从缓存中获取设备状态当作设备真实状态。在某些场景下该值并不是设备真实的状态，如HTTP短链接接入时，可能需要开发者调用接口定时获取设备状态等。此时平台提供的设备状态检测器并不满足业务逻辑，用户可以创建自定义类实现`org.jetlinks.core.device.DeviceStateChecker`接口作为自定义设备状态检测器，用户自定义设备状态检测器优先级高于平台默认设备状态检测器。

## 默认设备状态检测器
:::info
**<font style="color:#2F4BDA;">🌏</font>****<font style="color:#2F4BDA;"> 说明</font>**

设备使用的协议包中不存在自定义设备状态检测器时，设备状态检查会使用平台默认的检测器。

:::

核心代码

```java
package org.jetlinks.core.defaults;

@Slf4j
public class DefaultDeviceOperator implements DeviceOperator, StorageConfigurable {
    
    //默认检测逻辑
    private Mono<Byte> doCheckState() {
        return Mono
                .defer(() -> this
                        .getSelfConfigs(stateCacheKeys)
                        .flatMap(values -> {

                            //当前设备连接到的服务器
                            String server = values
                                    .getValue(connectionServerId)
                                    .orElse(null);

                            //设备缓存的状态
                            Byte state = values
                                    .getValue(stateKey)
                                    .orElse(DeviceState.unknown);

                            Mono<Byte> checker = handler
                                    .getDeviceState(server, Collections.singletonList(id))
                                    .map(DeviceStateInfo::getState)
                                    .singleOrEmpty()
                                    .defaultIfEmpty(state);
                             //当前缓存中没有server信息?
                            if (!StringUtils.hasText(server)) {
                                //网关设备ID
                                String parentGatewayId = values
                                        .getValue(DeviceConfigKey.parentGatewayId)
                                        .orElse(null);

                                if (getDeviceId().equals(parentGatewayId)) {
                                    log.warn(LocaleUtils.resolveMessage("validation.parent_id_and_id_can_not_be_same", parentGatewayId));
                                    return Mono.just(state);
                                }
                                boolean isSelfManageState = values.getValue(selfManageState).orElse(false);
                                //如果关联了上级网关设备则尝试给网关设备发送指令进行检查
                                if (StringUtils.hasText(parentGatewayId) && isSelfManageState) {
                                    return this
                                            .checkStateFromParent(parentGatewayId, state)
                                            .switchIfEmpty(checker);
                                }
                            }


                            return checker;

                        }));
    }
}
```

## 自定义设备状态检测器
1. 自定义类实现`org.jetlinks.core.device.DeviceStateChecker`接口

```java
package org.jetlinks.protocol.official.checker;

@Setter
@Getter
@AllArgsConstructor
public class CustomChecker implements DeviceStateChecker {

    @Override
    public @NotNull Mono<Byte> checkState(@NotNull DeviceOperator device) {

        return Mono.defer(()->{
            return doSomething(device);
        });
    }

    private <T> Mono<? extends Byte> doSomething(DeviceOperator device) {
        //自定义业务逻辑
         return Mono.just(DeviceState.online);
    }
}
```

2. 在官方协议包中添加设备状态检测器

```java
package org.jetlinks.protocol.official;

public class JetLinksProtocolSupportProvider implements ProtocolSupportProvider {

    @Override
        public Mono<CompositeProtocolSupport> create(ServiceContext context) {
            return Mono.defer(() -> {
                CompositeProtocolSupport support = new CompositeProtocolSupport();
                //添加自定义设备状态检测器
                support.setDeviceStateChecker(new CustomChecker());

                return Mono.just(support);
            });

}
```

1. 用户调用接口同步设备真实状态或查看设备详情时，会走平台的设备状态检测逻辑。
2. 获取设备状态值时，先到设备注册中心和使用设备状态检测器获取设备状态值，在使用设备状态检测器获取设备状态值时，若设备状态值不存在，则使用平台默认的状态检测器。
3. 使用平台默认的状态检测器获取设备状态值时，先判断当前设备使用的协议包中是否存在设备状态检测器，存在则使用协议包中检测器获取设备状态值，不存在则使用平台默认设备状态检测器从设备真实连接会话中获取状态值。未获取到设备状态值时，则默认设备为离线状态。
4. 获取到设备状态值后，合并状态结果集。在设备状态结果集中若设备检测器获取的状态值和设备注册中心获取的状态值不一致，则更新设备注册中心状态值为设备检测器中状态值，最后返回设备检测器中获取的设备状态值。

更新状态

1. DeviceGatewayHelper.handleDeviceMessage中处理状态变更调用DeviceSessionManager.compute创建或更新会话
2. AbstractDeviceSessionManager发布session注册或注销的事件
3. DeviceMessageConnector中监听session注册或注销事件并在onMessage中发布上下线消息
4. DeviceMessageBusinessHandler负责监听事件并更新数据库



查询状态

1. 前端通过WS请求设备实时状态，DashBoardSubscriptionProvider中订阅WS中topic
2. 平台在RealTimeDeviceStateDimension处理实时状态请求并调用设备的状态检测器
3. 设备会调用默认的checkstate判断设备在协议中是否实现DeviceStateChecker，实现则调用协议包中的检测器，否则调用默认状态检测器
4. 默认状态检测器会根据设备中的Broker去判断是单机还是集群，单机默认返回为空调用缓存中的状态，集群则判断本节点session是否包含设备，若查询返回为空则轮询集群节点
5. 若还未空将调用缓存中的状态



平台定时检测状态

1. 项目启动加载PersistenceDeviceSessionManager并执行初始化方法开启定时任务
2. 定时任务会在每30检查session 是否存活，不存活则移除localsession
3. 设备session过期将在AbstractDeviceSessionManager发送session注销事件
4. DeviceMessageConnector中监听session注册或注销事件并在onMessage中发布上下线消息
5. DeviceMessageBusinessHandler负责监听事件并更新数据库
