本文档主要介绍当平台下发指令给设备时，平台在一定时间内没有收到设备的确认消息，平台可以自动重发指令。

:::info
🌍 **<font style="color:#2F4BDA;">说明</font>**

本功能仅在2.1版本后支持，请更新至最新的2.1及其后版本进行使用。

:::

# 应用场景
1. 平台发送消息给设备后，监听响应流是否回复，若未回复响应超时异常平台自动重试该指令。
2. 平台发送消息给设备后，监听响应流设备回复了错误报文，平台重试设备指令。

# 前提条件
1. 下载JetLinks平台的源码，参考[获取源码](https://hanta.yuque.com/px7kg1/yfac2l/raspyc4p1asfuxks#R1bUq)。
2. 开发自定义协议包或使用[官方协议包](https://github.com/jetlinks/jetlinks-official-protocol)。

# 操作步骤
## 实现拦截器
1. `CustomMessageSenderInterceptor`实现`DeviceMessageSenderInterceptor`，主要实现核心方法`preSend(..)`和`doSend(..)`方法
    1. `preSend(..)`方法主要提供对`message`的更多处理，如本节添加重试参数需要在此方法内声明。
    2. `doSend(..)`方法在平台执行了发送逻辑后，拦截异常重构响应操作，本节内重构为重试下发指令。

```java
package org.jetlinks.protocol.official;

import org.jetlinks.core.device.DeviceOperator;
import org.jetlinks.core.message.DeviceMessage;
import org.jetlinks.core.message.interceptor.DeviceMessageSenderInterceptor;
import reactor.core.publisher.Flux;
import reactor.core.publisher.Mono;

public class CustomMessageSenderInterceptor implements DeviceMessageSenderInterceptor {

    @Override
    public Mono<DeviceMessage> preSend(DeviceOperator device, DeviceMessage message) {
		//默认使用父类处理逻辑，如需对DeviceMessage封装更多信息在此处编写自定义逻辑
        return DeviceMessageSenderInterceptor.super.preSend(device, message);
    }

    @Override
    public Flux<DeviceMessage> doSend(DeviceOperator device, DeviceMessage source, Flux<DeviceMessage> sender) {
		//默认使用父类处理逻辑
        return DeviceMessageSenderInterceptor.super.doSend(device,source,sender);
    }

}

```

## 添加拦截器支持
```java
@Override
public Mono<CompositeProtocolSupport> create(ServiceContext context) {
	...
	support.addMessageSenderInterceptor(new CustomMessageSenderInterceptor());
	...
}
```

## 核心类
1. `DeviceMessageSenderInterceptor`：无侵入方式拦截下发消息，在下发流程中封装参数/重构消息内容。

| **方法**                                                     | **返回值**            | **说明**                                                     |
| ------------------------------------------------------------ | --------------------- | ------------------------------------------------------------ |
| `preSend(DeviceOperator device, DeviceMessage message)`      | `Mono<DeviceMessage>` | 在消息发送前触发. 执行此方法后将使用返回值DeviceMessage进行发送到设备.用于添加消息头。 |
| `doSend(DeviceOperator device, DeviceMessage source, Flux<DeviceMessage> sender)` | `Flux<DeviceMessage>` | 执行发送时触发.设备无回复触发超时可抓取异常并实现自定义逻辑。 |


2.`DeviceMessageSendTaskInterceptor`：处理设备回复结果逻辑，根据本次任务响应结果更新实体类`DeviceMessageTaskEntity`数据，并在数据表`dev_message_task`中持久存储。

3.`DatabaseDeviceMessageTaskRepository`：本次任务数据存放到实体类`DeviceMessageTaskEntity`中，并在数据表`dev_message_task`持久存储。

## 添加消息重试头信息
:::warning
**<font style="color:#ECAA04;">💡</font>****<font style="color:#ECAA04;"> 注意</font>**

JetLinks平台超时时间默认10s，可在消息消息头中添加`Headers.timeout`，重新指定平台等待超时时长。更多消息头设置参数参考[统一设备消息定义——消息头](https://hanta.yuque.com/px7kg1/yfac2l/pzlxt6ef8urq0g3q#Vgs0U)。

:::

| **参数**        | 描述                                                         |
| --------------- | ------------------------------------------------------------ |
| `maxRetryTimes` | 最大重试次数                                                 |
| `useTask`       | 是否记录task记录，开启次参数后，下发指令消息记录存在数据表`dev_message_task`中。 |


```java
@Slf4j
@NoArgsConstructor
public class CustomMessageInterceptor implements DeviceMessageSenderInterceptor {

    private int totalRetry = 3; //最大重试次数

    public Mono<DeviceMessage> preSend(DeviceOperator device, DeviceMessage message) {
            FunctionInvokeMessage functionInvokeMessage = (FunctionInvokeMessage) message;
            //添加请求头  最大重试次数
            message.addHeader(PropertyConstants.maxRetryTimes.getKey(), totalRetry);
            //添加请求头参数 记录task记录
            message.addHeader(PropertyConstants.useTask, true);
            //添加自定义超时时间，不添加平台默认10s超时
            message.addHeader(Headers.timeout, TimeUnit.SECONDS.toMillis(40));
            return Mono.just(message);
    }
	
	...
}
```

## 重构重试逻辑
平台下发指令给设备，超过规定时间设备未回复时，平台自动进行消息重发。

```java
package org.jetlinks.costom.protocol;

import lombok.extern.slf4j.Slf4j;
import org.jetlinks.core.defaults.DefaultDeviceMessageSender;
import org.jetlinks.core.device.DeviceOperator;
import org.jetlinks.core.enums.ErrorCode;
import org.jetlinks.core.exception.DeviceOperationException;
import org.jetlinks.core.message.DeviceMessage;
import org.jetlinks.core.message.function.FunctionInvokeMessage;
import org.jetlinks.core.message.interceptor.DeviceMessageSenderInterceptor;
import org.jetlinks.pro.PropertyConstants;
import reactor.core.publisher.Flux;
import reactor.core.publisher.Mono;

import java.util.concurrent.TimeoutException;

@Slf4j
@NoArgsConstructor
public class CustomMessageInterceptor implements DeviceMessageSenderInterceptor {

    private int totalRetry = 3; //最大重试次数

    ...

    /**
     * 设备未回复时可在doSend中抓取异常，进行自定义逻辑处理
     */
    public Flux<DeviceMessage> doSend(DeviceOperator device, DeviceMessage source, Flux<DeviceMessage> sender) {
        return DeviceMessageSenderInterceptor.super
            .doSend(device, source, sender)
            // 本节以超时异常为示例，开发者可修改此处处理自有业务逻辑的异常类型。
            .onErrorResume(DeviceOperationException.class, err -> {
                if (err.getCode() == ErrorCode.TIME_OUT) {
                   //封装自定义逻辑，若需要对报文进行修改则需要在此处添加逻辑处理。
                    //若不需要对报文进行处理，则此处不需要写逻辑处理
                }
                return Mono.error(err);
            });
    }
}

```
