本文主要介绍JetLinks物联网平台的协议包中实现设备自注册功能，描述了设备自注册的应用场景以及实现步骤。

:::warning
**<font style="color:#ECAA04;">💡</font>****<font style="color:#ECAA04;"> 注意</font>**

MQTT直连接入的设备只支持网关子设备自注册。

:::

# 应用场景
当需要部署大量物联网设备时，手动注册每个设备将是一项繁琐的任务。设备自注册功能可以使设备在连接到平台后自动完成注册，减轻了管理和配置的负担。

# 实现原理
自定义协议包将消息解析为`DeviceRegisterMessage`，并设置`header`参数，如下表：

| **参数**        | **类型**           | **必选** |
| --------------- | ------------------ | -------- |
| `productId`     | String             | 是       |
| `deviceName`    | String             | 是       |
| `configuration` | Map<String,Object> | 否       |


平台将自动添加设备信息到设备实例中。

:::info
**<font style="color:#2F4BDA;">🌏</font>****<font style="color:#2F4BDA;"> 说明</font>**

**<font style="color:#DF2A3F;">如果是注册子设备，则解析为 </font>**`**<font style="color:#DF2A3F;">ChildDeviceMessage<DeviceRegisterMessage></font>**`**<font style="color:#DF2A3F;">即可。</font>**

+ 必须设置注册消息的`productId`和`deviceName`，否则注册失败抛出`设备不存在异常`。
+ `configuration`为设备的自定义配置信息，后续可通过`DeviceOperator#getSelfConfig()`来获取该参数。

:::

```java
ChildDeviceMessage childDeviceMessage = new ChildDeviceMessage();
childDeviceMessage.setDeviceId("网关设备ID");
DeviceRegisterMessage registerMessage = new DeviceRegisterMessage();
registerMessage.setDeviceId("网关子设备ID");
registerMessage.setMessageId(String.valueOf(SnowflakeIdGenerator.getInstance().nextId()));
Map<String,Object> headers = new HashMap<>();
headers.put(productId,"子设备对应的产品ID"); //需要注册的产品id
headers.put(deviceName,"子设备名称"); //需要注册的设备名称
registerMessage.setHeaders(headers);

childDeviceMessage.setChildDeviceMessage(registerMessage);
```

# 操作步骤
:::info
**<font style="color:#2F4BDA;">🌏</font>****<font style="color:#2F4BDA;"> 说明</font>**

以下操作步骤用官方协议中Tcp协议作为示例

:::

1. 在`TcpDeviceMessageCodec`中编写设备自注册逻辑

```java
public static final String PRODUCT_ID = "productId";

public static final String DEVICE_NAME = "deviceName";

private Mono<DeviceMessage> handleLogin(ByteBuf payload, MessageDecodeContext context) {
        DeviceMessage message = BinaryMessageType.read(payload);
        if (message instanceof DeviceOnlineMessage) {
            String token = message
                    .getHeader(BinaryDeviceOnlineMessage.loginToken)
                    .orElse(null);

            String deviceId = message.getDeviceId();
            return context
                    .getDevice(deviceId)
                    .flatMap(device -> device
                        .getConfig(CONFIG_KEY_SECURE_KEY)
                        .flatMap(config -> {
                            if (Objects.equals(config.asString(), token)) {
                                return ack(message, AckCode.ok, context)
                                    .thenReturn(message);
                            }
                            return Mono.empty();
                        }))
                    .switchIfEmpty(Mono.defer(()->{
                        DeviceRegisterMessage registerMessage = new DeviceRegisterMessage();
                        registerMessage.setDeviceId(deviceId);
                        registerMessage.setMessageId(String.valueOf(SnowflakeIdGenerator.getInstance().nextId()));
                        Map<String,Object> headers = new HashMap<>();
                        headers.put(PRODUCT_ID,"10003"); //需要注册的产品id
                        headers.put(DEVICE_NAME,"设备名称"); //需要注册的设备名称
                        registerMessage.setHeaders(headers);
                        return Mono.just(registerMessage);
                    }));

        } else {
            return ack(message, AckCode.noAuth, context);
        }
    }
```

:::warning
**<font style="color:#ECAA04;">💡</font>****<font style="color:#ECAA04;"> 注意</font>**

**<font style="color:#DF2A3F;">开启设备数据权限情况下，协议包内返回了自注册消息时：</font>**

1. **<font style="color:#DF2A3F;">平台默认会将当前注册的设备资产直接绑定在维度上。</font>**
+ **<font style="color:#DF2A3F;">直连设备：默认根据产品的数据权限（数据可见维度）进行绑定。</font>**
+ **<font style="color:#DF2A3F;">子设备：子设备绑定父设备的数据权限信息。</font>**
2. **<font style="color:#DF2A3F;">忽略资产绑定</font>**
+ **<font style="color:#DF2A3F;">在设备注册消息中</font>**`**<font style="color:#DF2A3F;">addHeader("ignoreBinding",true)</font>**`**<font style="color:#DF2A3F;">将忽略当前设备所属产品的数据权限操作。</font>**
+ **<font style="color:#DF2A3F;">在子设备注册消息中</font>**`**<font style="color:#DF2A3F;">addHeader("ignoreBinding",true)</font>**`**<font style="color:#DF2A3F;"> 将忽略绑定父设备的数据权限信息。</font>**

:::
